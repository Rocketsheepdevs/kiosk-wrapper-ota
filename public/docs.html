Kiosk Wrapper Web Integration

How a web app loaded inside kiosk_wrapper talks to the printer, NFC reader, and camera.

Overview

The kiosk_wrapper is a Flutter app that loads your web page in a full-screen WebView and exposes a JavaScript API for the kiosk hardware:

  • MASUNG MS-FPT201P 58 mm thermal printer (32 chars/line)
  • SYC ID & IC USB RFID reader (acts as an HID keyboard)
  • JSK USB camera for QR / barcode scans

All hardware access goes through a single global: window.kiosk. Method calls return Promises. Push events (card taps) arrive as DOM CustomEvents on window and via an internal event bus.

Minimum wrapper version — the API below requires kiosk_wrapper v1.0.8 or later. See version notes.

Detect the bridge

Before calling any hardware, confirm you're actually running inside the wrapper: 

if (window.kiosk && window.kiosk.isKiosk) {
  // Running inside kiosk_wrapper — hardware available
} else {
  // Running in a regular browser — no hardware
}

Drop-in JS helper

Copy-paste at the top of your kiosk web app for the typed wrapper:

const kiosk = {
  ready:   () => !!(window.kiosk && window.kiosk.isKiosk),
  print:   (text) => window.kiosk.print(text),
  printBytes: (base64) => window.kiosk.printBytes(base64),
  scanQr:  () => window.kiosk.scanQr(),
  lastCard:() => window.kiosk.lastCard(),
  onCard:  (cb) => window.kiosk.on('card', cb),
};

Text Print

The simplest case — call window.kiosk.print(text) with any string.

await window.kiosk.print(
  "*** ORDER #1234 ***\n" +
  "Item                100.00\n" +
  "Tax                  12.00\n" +
  "-------------------------------\n" +
  "TOTAL              112.00\n"
);

The wrapper automatically wraps your text with:

  • ESC @ — printer reset / initialize
  • Your text in UTF-8
  • ESC d 4 — feed 4 lines
  • GS V B 0 — full cut
Width: the MASUNG is 58 mm = 32 characters per line in the default font. Lines longer than 32 chars wrap. Use the helpers below to lay out columns.

Layout helpers (32-char width)

const W = 32;

function pad(left, right) {
  const gap = Math.max(1, W - left.length - right.length);
  return left + ' '.repeat(gap) + right;
}

function center(text) {
  const p = Math.max(0, Math.floor((W - text.length) / 2));
  return ' '.repeat(p) + text;
}

const hr = (ch = '-') => ch.repeat(W);

const receipt = [
  center('KAISER MEDICAL'),
  hr('='),
  `Date: 2026-05-21 10:30`,
  hr(),
  pad('Consultation', '500.00'),
  pad('Lab CBC',      '350.00'),
  hr(),
  pad('TOTAL', '850.00'),
  hr('='),
  '',
].join('\n');

await window.kiosk.print(receipt);

ESC/POS Formatting

You can embed ESC/POS control bytes directly in the text. The wrapper passes bytes < 0x80 through unchanged, so most formatting commands work inline. 

const ESC = '\x1B', GS = '\x1D';

const alignLeft   = ESC + 'a' + '\x00';
const alignCenter = ESC + 'a' + '\x01';
const alignRight  = ESC + 'a' + '\x02';
const boldOn      = ESC + 'E' + '\x01';
const boldOff     = ESC + 'E' + '\x00';
const bigOn       = GS  + '!' + '\x11';  // 2× wide, 2× tall
const bigOff      = GS  + '!' + '\x00';

await window.kiosk.print(
  alignCenter + bigOn + 'KAISER\n' + bigOff +
  alignLeft +
  boldOn + 'Patient:' + boldOff + ' Juan Cruz\n' +
  'Date: 2026-05-21\n'
);
StyleCommand bytes
Left / center / right alignESC a 0 / 1 / 2
Bold on / offESC E 1 / 0
Double size on / offGS ! 0x11 / 0x00
Initialize / resetESC @ (wrapper does this for you)

QR Codes (Raster Bitmap)

The MASUNG doesn't render ESC/POS QR commands (GS ( k). To print QR codes, generate the bitmap in JavaScript and send it via printBytes.

Load qrcode-generator:

<script src="https://cdn.jsdelivr.net/npm/qrcode-generator@1.4.4/qrcode.min.js"></script>

Build the raster and send:

function qrRasterBytes(text, scale = 6) {
  const qr = qrcode(0, 'M');
  qr.addData(text);
  qr.make();
  const n  = qr.getModuleCount();
  const w  = n * scale, h = n * scale;
  const bpr = Math.ceil(w / 8);
  const px  = new Uint8Array(bpr * h);
  for (let y = 0; y < h; y++) {
    for (let x = 0; x < w; x++) {
      if (qr.isDark(Math.floor(y/scale), Math.floor(x/scale))) {
        px[y*bpr + (x>>3)] |= 1 << (7 - (x & 7));
      }
    }
  }
  const header = new Uint8Array([
    0x1B, 0x61, 0x01,                       // ESC a 1 — center
    0x1D, 0x76, 0x30, 0x00,                 // GS v 0 m=normal
    bpr & 0xFF, (bpr >> 8) & 0xFF,        // xL xH (bytes per row)
    h   & 0xFF, (h   >> 8) & 0xFF,        // yL yH (height in dots)
  ]);
  const out = new Uint8Array(header.length + px.length);
  out.set(header, 0);
  out.set(px, header.length);
  return out;
}

function bytesToBase64(arr) {
  let s = '';
  for (let i = 0; i < arr.length; i++) s += String.fromCharCode(arr[i]);
  return btoa(s);
}

const raster = qrRasterBytes('https://kaisermed.com/r/ABC123');
await window.kiosk.printBytes(bytesToBase64(raster));
To print text + QR + text as one receipt without intermediate cuts, concatenate everything (text bytes + raster bytes + text bytes) into a single Uint8Array, base64 it, and call printBytes once. Multiple separate print/printBytes calls each get their own init+cut.

Raw bytes — any ESC/POS payload

window.kiosk.printBytes(base64) sends any byte sequence to the printer. The wrapper still prepends ESC @ and appends feed + cut, but the bytes in between are passed through verbatim. Use this for barcodes, bitmaps, custom commands.

NFC Card Scans push

The SYC reader sends card UIDs to the wrapper, which forwards them to the page automatically. Subscribe three ways — use whichever fits your code.

Idiomatic — DOM event listener

window.addEventListener('kiosk:card', (e) => {
  const uid = e.detail;
  console.log('card:', uid);
  // look up user, log in, etc.
});

Helper API

window.kiosk.on('card', (uid) => {
  console.log('card:', uid);
});

Legacy single callback (back-compat)

window.onKioskCardScan = (uid) => {
  console.log('card:', uid);
};

On-demand fetch (no callback)

const { value } = await window.kiosk.lastCard();
// value === most recently scanned UID, or null if none yet
The reader detection algorithm distinguishes a card swipe from human typing by measuring inter-keystroke timing (< 80 ms between keys = card burst). Card UIDs are pushed only when the burst ends with Enter — typing slowly into a form won't trigger card events.

Camera QR Scan

One-shot — call scanQr(), the camera page opens, user points at a QR/barcode, the value comes back:

const result = await window.kiosk.scanQr();
if (result.ok) {
  console.log('scanned:', result.value);
} else if (result.cancelled) {
  console.log('user cancelled');
}

Supported formats: QR, EAN-13, Code-128, UPC, ITF, Data Matrix, Aztec, PDF417 (Google ML Kit).

Errors & Troubleshooting

SymptomCauseFix
ok: false, error: "No USB devices found" Printer disconnected / power off Reseat USB cable, reboot kiosk
ok: false, error: "USB permission denied" User dismissed Android USB dialog Replug printer, tap "Allow" + "Use by default"
Print sends but printer ignores certain commands MASUNG doesn't implement that ESC/POS function Use raster (printBytes) instead
Text wraps unexpectedly Line longer than 32 chars Use pad()/center() helpers @ W=32
QR doesn't scan Scale too small Increase scale (try 5–7); raster width must be ≤ 384 dots
Card events never fire Listener registered after the scan Register listeners as early as possible (in <head>)
